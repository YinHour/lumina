# 研发周报 (截至 2026.4.26)

## 本周工作总结

本周主要围绕“笔记本密码保护与用户访问控制”、“来源(Sources)列表页面的重构与优化”、“来源安全删除与精细化管理”、“服务性能与稳定性优化”以及“新一代文档解析工具的性能验证”五个核心方向展开。

以下是按照逻辑分配的每日工作量梳理：

### Day 1: 来源列表重构与基础 Bug 修复
*   **重构来源列表分页机制**：解决了当文档数量过多时列表显示不全（仅显示约30个）且刷新无效的问题，将原有的无限滚动加载方式彻底重构为传统的页码分页（每页30篇），提升了长列表的渲染性能。
*   **修复 i18n 代理导致的渲染报错**：定位并修复了前端高频出现的 `TypeError: undefined is not an object (evaluating 't.sources.type.file')` 报错，通过在渲染循环外缓存翻译键值，避免了对 i18next Proxy 的过度频繁访问。
*   **修复“添加现有来源”搜索匹配问题**：优化了后端的 SurrealQL 查询逻辑，将精确匹配改为 `string::contains` 子串匹配，解决了输入包含在名称中的字却检索不到源文件的问题。

### Day 2: 笔记本密码保护与访问控制
*   **数据库 Schema 升级**：编写并应用了第 16 号数据库迁移脚本（`16.surrealql`），为 `notebook` 表安全地新增了 `password` 和 `creator_name` 字段，并确保兼容旧有的无密码笔记本数据。
*   **笔记本创建与展示优化**：在创建笔记本的表单中加入了可选的密码设置和创建人名称填写功能。在笔记本卡片列表的合适位置（更新时间旁）增加了创建人展示，并为加密笔记本添加了直观的“锁定”图标。
*   **严格的前端访问拦截**：实现了全屏的密码输入拦截页面。移除了最初的 `sessionStorage` 缓存逻辑，确保用户**每次**访问加密笔记本页面时都必须重新输入密码，增强了隐私保护。

### Day 3: 全局来源安全删除与系统级密码管理
*   **引入系统级管理员密码**：在前端环境变量（`.env.local`）中配置了 `NEXT_PUBLIC_MASTER_NOTEBOOK_PASSWORD`，该超级密码可以绕过单个笔记本的密码直接访问任何加密笔记本。
*   **全局来源删除的安全加固**：在“所有来源”页面恢复了删除按钮，但为其加上了管理员权限校验。重写了删除确认交互，使用美观的系统级 `Dialog` 组件替代了简陋的 `window.prompt`。
*   **修复密码弹窗的键盘事件冲突**：深入排查并修复了在密码输入框中输入时，由于底层列表的键盘导航快捷键（上下方向键、回车键）拦截事件，导致输入框莫名失去焦点、内容清空或光标消失的严重交互 Bug。同时支持了在输入框内直接按回车键确认删除。

### Day 4: 来源精细化管理与引用统计
*   **来源导入位置追踪**：编写并应用了第 17 号数据库迁移脚本，在 `source` 表中新增 `origin_notebook_id` 字段，用于精准记录每个来源最初是由哪个笔记本上传导入的。
*   **笔记本内删除逻辑智能化**：基于导入位置重构了笔记本内部的来源操作逻辑。如果文件是**在当前笔记本直接上传的**，则允许“删除来源”；如果是**从现有来源添加的**（即公共资产），则仅允许“从笔记本移除”（解绑），有效防止了对公共文件的误删。
*   **新增来源引用热度统计**：优化了后端的聚合查询逻辑，在“所有来源”页面的表格中新增了一列“引用次数”，直观展示每个来源文件当前被多少个笔记本关联和引用。

### Day 5: 服务并发性能研究与新一代解析工具验证
*   **服务并发性能研究**：若想解决多用户并发，需通过引入数据库单例长连接/连接池（解决连接雪崩与并发报错）、用 SurrealDB 替代 SQLite 存储 LangGraph 状态（解决单文件读写死锁）、按请求显式传递 API Key 参数（解决全局环境变量串键污染）、采用 UUID 策略保存文件（解决文件上传竞态覆盖）、引入分布式任务防抖锁（解决昂贵 AI 算力的重复消耗），以及追加轻量级租户标识与前端轮询（解决数据越权访问与幽灵脏读），从而在“网络、内存、文件、数据库”四个物理维度彻底实现请求隔离与状态安全地彻底消除资源争抢。
*   **新一代文档解析工具的性能验证**：测试并完善了新一代的文档解析引擎MinerU，针对复杂的 PDF（如包含图表、多栏排版的实验报告）进行了提取准确率、解析速度的验证评估。

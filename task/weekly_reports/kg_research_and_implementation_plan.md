# 科研文献知识图谱（Knowledge Graph）引入调研与实施计划书

## 一、 背景与痛点分析

当前 Open-Notebook 系统已具备基于纯向量检索（Vector RAG）的多文档问答能力。但在针对**科研论文和实验报告**的特定场景深度测试中，我们发现了现有纯向量检索的几个致命短板：

1. **上下文断裂**：传统的文本切块（Chunking）极易破坏科学实验中“条件-材料-结果”之间的长程关联。
2. **多跳推理（Multi-hop Reasoning）能力差**：当用户提问“在 X 温度下使用 Y 材料的转化率是多少？”时，向量检索往往只能找回分别包含 X、Y 和转化率的零散文本块，大模型极易产生“缝合怪”式的幻觉，把不相干的实验结果强行拼接。
3. **要素关系模糊**：科学文献的核心是严谨的实体关联。纯文本向量无法表达“催化剂A”是作为“实验2”的反应物，还是仅仅在综述段落中被提及。

为此，我们计划在系统中引入**知识图谱（Knowledge Graph）增强**，构建 Hybrid RAG 架构。

---

## 二、 核心架构设计方案

### 2.1 面向科研的图建模：引入“枢纽节点（Hub Node）”
传统的知识图谱采用简单的 `(实体A)-[关系]->(实体B)` 三元组模型，这在处理复杂实验时会导致逻辑崩塌（例如无法区分不同温度下的同一组反应）。

**计划方案：**
我们将独创性地引入 `EXPERIMENT`（实验）作为图谱的枢纽节点。所有相关的实验要素均辐射状地连接到该枢纽：
- `(MATERIAL) -[USES_MATERIAL]-> (EXPERIMENT)`
- `(CONDITION) -[HAS_CONDITION]-> (EXPERIMENT)`
- `(EXPERIMENT) -[YIELDS_RESULT]-> (RESULT)`
通过这种超关系（Hyper-relational）建模，确保复杂的多元组关系在图谱中被原子化、结构化地隔离，彻底杜绝条件错配。

### 2.2 抽取策略：大模型驱动的无切块（Chunk-free）全篇抽取
传统 NLP 工具难以精准提取学术黑话。我们将完全依托强力大语言模型（如 Qwen/DeepSeek）的超长上下文窗口能力。

**计划方案：**
- **取消文本切块**：将整篇论文合并为单一的长字符串（放宽 `max_tokens` 限制至 60k+），单次送入大模型，从根本上解决跨段落指代不清的问题。
- **防幻觉数值过滤（Ambiguity Filter）**：通过精心设计的 Prompt 约束，禁止大模型将孤立的数字（如页码、随机参数）抽取为实体。所有数值必须作为核心实体或关系的属性（如 `description` 字段）存在，提升图谱纯净度。

### 2.3 底层存储基建：SurrealDB 原生图引擎
Open-Notebook 现已使用 SurrealDB，我们将充分挖掘其被闲置的原生图数据库（Graph DB）潜力。

**计划方案：**
- 建立专门的 `kg_entity`（实体表）和 `kg_relation`（边表）。
- 利用 SurrealDB 的原生 `RELATE` 语句建立实体关联，其图遍历查询效率远超传统关系型数据库的 JOIN 操作。
- **实体消歧机制（Entity Resolution）**：开发统一的 `slugify` 算法生成一致性 ID，并在入库时采用 `UPSERT` 语法，实现跨文档同名实体知识的自动合并与扩展。

### 2.4 检索融合：无缝嵌入 Ask 模式的 Hybrid RAG
知识图谱的引入不应增加用户的交互负担，而应作为一种底层的“静默强化”。

**计划方案：**
- 在现有的 Ask 模式中，当 Agent 规划出检索关键词后，底层将**并行**发起两种搜索：
  1. 传统的向量相似度搜索（Vector Search）。
  2. **图谱子图扩展（Subgraph Expansion）**：首先利用 BM25 全文检索匹配到图谱中的入口节点，然后执行 1-hop（单跳）关系遍历，召回结构化的相邻实体与边描述。
- 最终，将“高维语义片段”与“严谨的结构化知识路径”融合组装成超级 Context，交由应答模型进行最终决策。
